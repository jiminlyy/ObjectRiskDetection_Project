# 거리 측정 및 위험 판단 전략

## 1. 교수님 요구사항에 대한 정리

**"거리를 정확하게 어떻게 측정할 건가?"**
→ LiDAR 센서, 바운딩 박스 역산, TTC(충돌 예상 시간) 계산을 거리 구간별로 조합하여 측정합니다.

**"몇 미터에서부터 위험이라고 판단할 건가?"**
→ 위험 요소의 이동 속도에 따라 3초 전 경고가 가능한 거리를 기준으로 설정합니다.

**"부딪히기 1초 전에 알면 어떡할 건가?"**
→ TTC를 실시간으로 계산하여 3초 전부터 단계별 경고를 시작합니다. 1초 전이 아니라 3초 전에 이미 경고가 완료됩니다.

---

## 2. 거리 측정 — 3단계 융합 전략

YOLO 모델 단독으로는 거리 측정이 불가능합니다. 따라서 거리 구간별로 서로 다른 측정 방법을 조합하여 사용합니다.

### 근거리 (0 ~ 5m): LiDAR 센서 정밀 측정

iPhone Pro 시리즈에 탑재된 LiDAR 센서를 활용합니다. LiDAR는 레이저를 발사하여 반사 시간을 측정하는 방식으로, 실외 환경에서 0.5m ~ 5m 범위에서 ±0.3m 이내의 정밀도를 제공합니다.

동작 방식은 다음과 같습니다. YOLO가 객체의 화면 좌표를 제공하면, iOS ARKit의 depth map에서 해당 좌표의 거리값을 읽어옵니다. 예를 들어 YOLO가 화면의 (0.5, 0.6) 위치에 차량을 감지하면, LiDAR depth map에서 동일 좌표의 값이 3.2m라면 "차량이 3.2m 거리에 있다"고 판단합니다.

이 범위는 시각장애인이 즉각적인 회피 행동이 필요한 긴급 구간이므로, 가장 정밀한 측정이 필요합니다.

### 중거리 (5 ~ 15m): 바운딩 박스 크기 기반 역산 추정

LiDAR 유효 범위를 벗어나는 구간에서는 핀홀 카메라 모델 원리를 활용합니다. 물체의 실제 크기를 사전에 등록해두고, 바운딩 박스의 화면 내 비율로 거리를 역산합니다.

사전 등록 물체 크기는 다음과 같습니다. 사람은 약 1.7m, 승용차는 약 1.5m 높이, 버스는 약 3.2m, 트럭은 약 3.5m, 자전거(탑승 시)는 약 1.0m, 오토바이(탑승 시)는 약 1.1m입니다.

계산 예시로, 사람이 화면 높이의 30%를 차지하면 약 5.7m, 50%를 차지하면 약 3.4m, 80%를 차지하면 약 2.1m로 추정됩니다.

이 방법의 오차는 ±1~2m 수준이지만, "약 10m 거리에 차량 접근 중"과 같은 수준의 안내에는 충분합니다.

### 원거리 (15m 이상): YOLO + Object Tracking

15m 이상에서는 정밀한 거리 측정보다 접근 여부 판단이 더 중요합니다. YOLOv8 내장 ByteTrack 알고리즘으로 동일 객체를 프레임 간 추적하고, 바운딩 박스 크기의 변화율을 통해 "접근 중" 또는 "이탈 중"을 판단합니다.

바운딩 박스 면적이 프레임마다 증가하면 접근, 감소하면 이탈로 판단합니다. 증가율이 클수록 빠르게 접근하는 것이므로, 이를 기반으로 TTC(충돌 예상 시간)를 계산합니다.

---

## 3. TTC (Time To Collision) — 충돌 예상 시간 계산

TTC는 현재 접근 속도가 유지될 때 위험 거리에 도달하기까지 남은 시간입니다. 정확한 미터 단위 거리를 몰라도 바운딩 박스 변화율만으로 계산 가능합니다.

계산 공식은 다음과 같습니다.

```
TTC(초) = (위험 임계값 - 현재 바운딩 박스 면적) / 면적 변화율 / FPS

예시:
현재 바운딩 박스 면적 = 화면의 7%
면적 변화율 = 프레임당 +1%
FPS = 30
위험 임계값 = 화면의 30% (매우 가까움)

TTC = (0.30 - 0.07) / 0.01 / 30 = 0.77초 후 위험 범위 진입
```

이 값을 기반으로 3단계 경고를 발생시킵니다.

| TTC | 경고 수준 | 알림 방식 | 사용자 행동 |
|-----|---------|---------|----------|
| 3초 이상 | 주의 | 음성 안내 "정면에 차량 접근 중" | 인지 및 준비 |
| 1.5 ~ 3초 | 경고 | 음성 + 진동 "차량 가까이 접근 중, 주의하세요" | 방향 전환 준비 |
| 1.5초 이내 | 긴급 | 비프음 + 강한 진동 | 즉시 회피 |

보행자의 평균 반응 시간은 0.7 ~ 1초입니다. 3초 전부터 경고를 시작하면, 사용자가 음성을 듣고(0.5초), 상황을 인지하고(0.5초), 회피 행동을 시작하기까지(1초) 총 2초가 소요되어도 1초의 여유가 남습니다.

따라서 "부딪히기 1초 전에 알면 어떡하냐"에 대한 답은, 1초 전이 아니라 3초 전에 이미 첫 경고가 시작되며, 1.5초 전에는 긴급 경고로 전환되어 충분한 회피 시간을 확보한다는 것입니다.

## 4. 위험 요소별 감지 거리 및 대응 전략

각 위험 요소의 이동 속도를 기준으로, 3초 전 경고에 필요한 최소 감지 거리를 산정합니다.

| 위험 요소 | 예상 속도 | 3초간 이동 거리 | 감지 방법 | 경고 방식 |
|----------|---------|-------------|---------|---------|
| 보행자 | 4 km/h | 3.3m | LiDAR 정밀 측정 | 음성 안내 |
| 자전거 | 15 km/h | 12.5m | 역산 추정 + TTC | 음성 + 진동 |
| 킥보드 | 20 km/h | 16.7m | 역산 추정 + TTC | 음성 + 진동 |
| 차량 (골목) | 30 km/h | 25.0m | YOLO + Tracking + TTC | 비프음 + 진동 |
| 차량 (도심) | 50 km/h | 41.7m | YOLO + Tracking + TTC | 비프음 + 진동 |

보행자의 경우 3.3m이므로 LiDAR 유효 범위(5m) 내에서 정밀 감지가 가능합니다. 자전거와 킥보드는 12~17m로 바운딩 박스 역산과 TTC로 대응합니다. 차량은 25~42m로 YOLO 탐지 + Tracking이 주 수단이며, 차량은 바운딩 박스가 크기 때문에 30m 이상에서도 YOLO가 안정적으로 감지합니다.

## 5. 위험 요소 분류 체계

일상생활에서 시각장애인이 마주치는 위험 요소를 두 가지로 분류합니다.

### 동적 위험 요소 (이동하는 물체)

차량, 오토바이, 자전거, 킥보드, 보행자가 해당합니다. 이들은 YOLO로 탐지하고, Object Tracking으로 프레임 간 추적하여 접근 여부와 TTC를 계산합니다. 동적 위험 요소는 방향(정면이 측면보다 위험), 거리(가까울수록 위험), TTC(충돌 예상 시간이 짧을수록 위험) 세 가지 기준으로 위험도를 종합 판단합니다.

### 정적 위험 요소 (도로 환경)

차도와 인도의 경계가 해당합니다. 세그멘테이션 모델(DeepLabV3)로 도로(road)와 인도(sidewalk)를 픽셀 단위로 구분하고, 사용자가 차도에 진입하면 즉시 진동으로 경고합니다. 이 기능은 시각장애인 교통사고의 주요 원인인 차도 무의식 진입을 사전에 방지합니다.

## 6. 시스템 통합 파이프라인

```
매 프레임:
├── YOLO → 객체 탐지 (무엇이 어디에 있는가)
├── Tracking → 동일 객체 추적 (같은 차가 이동 중인가)
├── 거리 측정:
│   ├── 0~5m → LiDAR 정밀 거리
│   ├── 5~15m → 바운딩 박스 역산 거리
│   └── 15m+ → 존재 여부 + 접근/이탈만 판단
├── TTC 계산 → 충돌까지 남은 시간
└── 경고 출력:
    ├── TTC > 3초 → 음성 "정면에 차량 접근 중"
    ├── TTC 1.5~3초 → 음성 + 진동 "차량 가까이 접근 중, 주의하세요"
    └── TTC < 1.5초 → 비프음 + 강한 진동

4프레임당 1회:
└── 세그멘테이션 → 도로 경계 판단 → 이탈 시 진동 경고
```

## 7. 정량적 성능 목표

| 항목 | 목표 수치 | 측정 방법 |
|------|---------|---------|
| 근거리 거리 측정 정확도 | ±0.3m (5m 이내) | LiDAR 기준 |
| 중거리 거리 추정 정확도 | ±2m (5~15m) | 바운딩 박스 역산 |
| 객체 감지 → 음성 안내 | 0.5초 이내 | 프레임 처리 + TTS 지연 |
| 긴급 상황 → 비프음 | 0.1초 이내 | 사전 로드된 사운드 재생 |
| 도로 이탈 → 진동 | 0.2초 이내 | 세그멘테이션 결과 즉시 반영 |
| 3초 전 경고 가능 범위 | 보행자 3.3m ~ 차량 42m | 위험 요소 속도별 |
| 전체 파이프라인 FPS | 20~30 FPS | 플래그십 스마트폰 기준 |

## 8. 핵심 메시지

본 시스템의 거리 측정은 단일 방법에 의존하지 않고, LiDAR(근거리), 바운딩 박스 역산(중거리), Object Tracking(원거리)을 거리 구간별로 조합하여 사용합니다. 이를 통해 각 방법의 한계를 상호 보완하며, 정밀한 거리 정보가 필요한 근거리에서는 LiDAR로, 사전 경고가 필요한 원거리에서는 Tracking + TTC로 대응합니다.

가장 중요한 것은 단순히 "차량이 있다"를 알려주는 것이 아니라, "차량이 몇 초 후에 위험 범위에 진입하는가"를 계산하여 사용자에게 충분한 회피 시간을 제공하는 것입니다. TTC 3초 전부터 단계별 경고를 시작함으로써, 보행자 반응 시간(0.7~1초)을 고려하더라도 최소 1초 이상의 회피 시간을 보장합니다.
